    Hardware, Kernels, and LLMs #

    Overview #

    This section establishes the background needed for the rest of the tutorial. We survey the landscape of AI accelerators — GPUs, TPUs, and AWS Trainium — build a mental model of the software stacks that turn model code into executable kernels, and introduce the main kernel programming interfaces (CUDA, Triton, and NKI). We then turn to the language models themselves: how they decode and how inference-time scaling works, followed by the two pillars of post-training used throughout the tutorial — supervised fine-tuning (SFT) for code generation and reinforcement learning (PPO and GRPO variants).

    Contents of this section #

    1.1 Overview of AI Hardware: GPUs, TPUs, and AWS Trainium #

    Surveys modern AI accelerators — GPUs, TPUs, and AWS Trainium — and the hardware features (compute engines, memory hierarchies, and interconnects) that kernels must target.

    1.2 Overview of AI Software Stacks #

    Builds a mental model of the AI software / ML compiler stack — from PyTorch programs through graph capture, compiler IRs, and kernel libraries down to accelerator machine code — and shows where hand-written or agent-written kernels become necessary.

    1.3 Overview of System Kernels: CUDA, Triton, and NKI #

    Introduces the kernel programming interfaces used in the tutorial — CUDA for NVIDIA GPUs, Triton for portable GPU kernels, and the Neuron Kernel Interface (NKI) for AWS Trainium.

    1.4 LLM Decoding and Inference-Time Scaling #

    Reviews how LLMs generate text through autoregressive decoding and how inference-time compute can be scaled to trade extra computation for higher-quality outputs.

    1.5 Supervised Fine-Tuning (SFT) for Code Generation #

    Covers supervised fine-tuning as the first stage of adapting base language models into capable code and kernel generators.

    1.6 RL Post-Training: PPO and GRPO Variants #

    Introduces reinforcement learning-based post-training — including PPO and GRPO variants — that aligns models toward generating correct and performant code.
